Identifiers:
⦁ An identifier is a name used for variables, methods, classes, etc., in Java.
⦁ It is made up of one or more characters.
⦁ First character must be:
A letter (a–z or A–Z)
A dollar sign ($)
An underscore (_)
⦁ Other characters (after the first) can be:
Letters (a–z or A–Z)
Digits (0–9)
Dollar sign ($)
Underscore (_)
⦁ The question mark (?) is not allowed in identifiers.
⦁ Java uses the Character.isJavaIdentifierStart() and Character.isJavaIdentifierPart() methods to check valid characters.

Rules:
Must begin with:
A letter (A–Z or a–z)
An underscore (_)
A dollar sign ($)
Can contain after the first character:
Letters (A–Z, a–z)
Digits (0–9)
Underscore (_)
Dollar sign ($)
Cannot be:
A Java keyword (like int, class, public, etc.)
A reserved literal (true, false, null)
A question mark (?) or any other special character
No length limit – identifiers can be long, but should be readable.
Case-sensitive – Variable, variable, and VARIABLE are all different.
Arrays:
A collection of data items, all of the same type, in which each item's position is uniquely designated by an integer.
Example: int[] numbers = {1, 2, 3, 4, 5};
Arrays can be single-dimensional or multi-dimensional.
Methods
A method in Java is a block of code that performs a specific task. It is used to operate on data (variables) and promote code reuse and organization.
A method is:
Group of Java statements used to perform operations on data.
Always declared inside a class.
Designed for code reuse, better organization, and maintainability.
Components: access-modifier: public, private, protected, or default (package-private).
returnType: The data type of the value the method returns (e.g., int, String, void).
methodName: The name of the method.
parameterList: A list of parameters passed to the method.
Advantages of Using Methods :
Code Reusability: Write once, use multiple times.
Modular Code: Easier to read and maintain.
Debugging: Simplifies finding and fixing bugs.
Testing: Each method can be tested individually.
Syntax:
<access-modifier> returnType methodName(parameterList) {
// Method body
